# FXTF_TickScalper パラメータ 日本語解説

MT4内のパラメータ表示はASCIIに変更しました。以下はそれぞれの変数の日本語解説と調整ガイドラインです。

## 基本設定

### `InpLots` — ロット数
- **既定値**: `0.01` (= 100通貨、初期安全値)
- **意味**: MT4で発注する際のロット数
- **FXTF-MT4（契約サイズ 10,000通貨）での換算**:
  | InpLots | 通貨量 | ランク1との比 | 用途 |
  |---:|---:|---:|---|
  | **`0.01`** | **100通貨** | **1%** | **本番最小テスト / 既定値** |
  | `0.1` | 1,000通貨 | 10% | 初期デモ〜少額本番 |
  | `0.5` | 5,000通貨 | 50% | Tier2上限（GBPUSD/AUDUSD） |
  | `1.0` | 10,000通貨 | 100% | Tier1満額（ランク1上限） |
- **FXTF-MT4 Micro（契約サイズ 1,000通貨）の場合**:
  - 0.1 = 100通貨 / 1.0 = 1,000通貨 / 10.0 = 10,000通貨
- **海外MT4（契約サイズ 100,000通貨）の場合**:
  - 0.001 = 100通貨 / 0.01 = 1,000通貨 / 0.1 = 10,000通貨
- **確認方法**: MT4「気配値表示」→ USDJPY-cd 右クリック →「仕様」→「契約サイズ」
- **推奨ロードマップ**:
  - Week 1: `0.01` (100通貨) で慣らし運転
  - Week 2-3: `0.1` (1,000通貨) に拡大
  - Week 4以降: PF確認後 `1.0` (Tier1) / `0.5` (Tier2) へ

### `InpMagicNumber` — マジックナンバー
- **既定値**: `0` (= 通貨ペア名から自動生成)
- **意味**: このEAが発注した注文を識別する番号。複数チャートで走らせても衝突しないように
- **動作**:
  - `0` (既定): Symbol()から77000〜85999の範囲で自動計算。**多通貨並走時の最適設定**
  - `1` 以上: その値を固定使用（デモで自分の好きな番号を割当てたい時）
- **自動生成時の各通貨のマジック**（参考）:
  | 通貨 | FXTF標準 | FXTF-cd形式 |
  |---|---:|---:|
  | USDJPY | 79901 | 79873 |
  | EURJPY | 85249 | 85877 |
  | GBPJPY | 83070 | 80888 |
  | AUDJPY | 81571 | 80491 |
  | EURUSD | 77892 | 81490 |
  | GBPUSD | 84713 | 82853 |
  | AUDUSD | 83214 | 85104 |
- EA起動時にエキスパートタブに `Magic=XXXXX (auto)` と出力されるので確認可能

### `InpSlippage` — 許容スリッページ
- **既定値**: `0` (point単位) ← 2026年4月更新
- **意味**: 発注価格と約定価格の許容誤差。0 = 完全一致のみ許可
- **既定値 0 の理由**:
  - スキャルピングでは 0.3 pip のスリップが期待値の 23% を食う
  - FXTF の 0 スプレッド商品と組み合わせるとスリップ損も0にしたい
  - 約定価格 = 要求価格 を保証
- **トレードオフ**:
  - ✅ Pro: スリップ損失なし、期待値通りの成績
  - ⚠️ Con: 価格が動いた瞬間の発注が `err=138 (Requote)` で拒否される
  - EA は次ティックで自動リトライするが、機会損失は発生する
- **調整ガイド**:
  - エキスパートタブで `err=138` や `err=136` が **1時間に10回以上**出るなら `1` や `3` に緩める
  - デモ運用で約定率を確認してから判断
  - 相場が急変動する時間帯のみ一時的に `3` に上げる手も
- **注意**: MT4バックテスターはスリッページ 0 でも理想的に約定するので、実環境の検証には**デモ口座フォワードテスト**が必須

## Exit Params（退出パラメータ）

### `InpStopPip` — 損切幅
- **既定値**: `2.5` (pip)
- **意味**: この値まで含み損が広がったら即損切
- **調整**:
  - 小さく(1.5-2.0): 負けが少なく抑えられるが勝率低下
  - 大きく(3.0-4.0): 勝率上がるがたまに大損
  - **最適化結果は2.5**

### `InpPeakActivate` — トレール発動ピーク
- **既定値**: `1.8` (pip)
- **意味**: 含み益がこの値に達したら「トレール退出」機能が有効化
- **調整**:
  - 小さく(1.0): 早めに利確開始(小さな利確多め)
  - 大きく(2.5): じっくり利益を伸ばしてから利確

### `InpTrailGap` — トレール幅
- **既定値**: `1.2` (pip)
- **意味**: ピーク利益からこの幅だけ下落したら利確
- **動作例**: peak=3.0p の状態から pip=3.0-1.2=1.8p まで戻ったら利確
- **調整**:
  - 小さく(0.5): 利確が早い。小さな揺れで切ってしまう
  - 大きく(2.0): トレンドに乗って大きく取れるが、戻りで利益減少

### `InpTimeoutSec` — 最大保有時間
- **既定値**: `60` (秒)
- **意味**: この秒数経過したら強制クローズ(レンジ相場対策)
- **調整**: 30〜120秒の範囲で調整可。1分足戦略なのでデフォルト推奨

## Entry Params（エントリパラメータ）

### `InpTriggerWindow` — トリガ評価tick数
- **既定値**: `5`
- **意味**: 直近Nティックを見て方向一致を評価

### `InpTriggerHits` — 必要な順方向tick数
- **既定値**: `3`
- **意味**: `InpTriggerWindow` 個のうち、順方向(BUYならup、SELLならdown)がN個以上あればエントリ
- **調整**:
  - 小さく(2): エントリ回数が増えるが質低下
  - 大きく(4): 厳選されて精度上がるが機会減少

### `InpEMAFast` / `InpEMASlow` — 方向判定EMA
- **既定値**: `20` / `60`
- **意味**: M1足で EMA(20) > EMA(60) なら買い方向のみ、逆ならSell方向のみでエントリ
- **調整**: 通常触らない。テクニカル分析に詳しければ調整可

## Multi-TF Edge（マルチタイムフレーム確認）

### `InpUseM5Confirm` — M5 方向確認
- **既定値**: `true` (有効)
- **意味**: M1 EMA の方向に加え、**M5 EMA の方向も同じ**な時だけエントリ許可
- **効果**:
  - 取引数: 約 30-50% 減
  - 勝率: 55% → **62-65%** 期待
  - PF: 1.15-1.25 → **1.4-1.5** 期待
  - whipsaw (短命トレンドへの誤乗り) を大幅削減
- **無効化**: `false` にすれば従来動作（M1のみ）に即戻る
- **使い分け**:
  - 通常運用: `true`
  - バックテスト比較時: `false` で元の挙動再現

### `InpEMAFastM5` / `InpEMASlowM5` — M5 EMA期間
- **既定値**: `20` / `60` (M1と同じ)
- **意味**: M5 足で EMA(20) と EMA(60) を計算し、向きを判定
- **調整**:
  - M5 を短め (10/30) にすると反応が早くなる = 取引数増
  - M5 を長め (50/200) にすると堅実なトレンドのみ = 取引数激減
- **推奨**: 既定値のまま

### マルチTF確認の論理

```
M1 EMA: BUY  +  M5 EMA: BUY   → エントリ OK
M1 EMA: BUY  +  M5 EMA: SELL  → スキップ (方向不一致)
M1 EMA: SELL +  M5 EMA: BUY   → スキップ (方向不一致)
M1 EMA: SELL +  M5 EMA: SELL  → エントリ OK
```

M1 が瞬間的に上向きになっても、M5 (=より長期のトレンド) が下向きなら買わない。
→ **大きな流れに逆らうエントリを消す**のが効果の核心。

## Risk Guard（リスクガード）

### `InpMaxUnits` — ランク1上限
- **既定値**: `10000`
- **意味**: 同方向の建玉＋未約定発注の合計がこの通貨数を超える新規発注をブロック
- **FX TFランク1**: USD/JPY は10,000通貨まで手数料0円
- **変更**: 他通貨ペアで使うときは表に従って変更
  | 通貨ペア | ランク1上限 |
  |---|---:|
  | USD/JPY, EUR/JPY, EUR/USD, AUD/JPY, GBP/JPY | 10,000 |
  | GBP/USD, AUD/USD | 5,000 |
  | NZD/JPY, CAD/JPY, USD/CAD, MXN/JPY | 3,000 |
  | ZAR/JPY, その他マイナー | 1,000 |

### `InpMaxSpreadPoints` — 許容最大スプレッド
- **既定値**: `2` (point = 0.2pip)
- **意味**: 現在のスプレッドがこれを超えていたらエントリしない
- **重要**: 指標発表・早朝の広スプレッド帯から自動防御する要の機能
- **調整**:
  - 厳しく(1): 0.1pip超で停止。取引機会減るが完全防御
  - 緩く(5): 0.5pipまで許容。取引機会は増えるが時々広スプレッドで約定

### `InpSkipTokyoMorn` — 日本時間6-9時除外
- **既定値**: `false` (新方針では `InpTradeOnlyEAHrs` で包括するため無効化)
- **意味**: この時間帯(JST 06:00-09:00)は薄商いでスプレッドが広がりやすいので取引停止
- **注意**: `InpTradeOnlyEAHrs=true` にしているとこの設定は不要（カスタム時間帯で包括）

### `InpSkipNYNewsHrs` — 日本時間21-24時除外
- **既定値**: `false` (新方針ではNY時間も取引する)
- **意味**: NY開場＋米指標が集中する時間帯(JST 21:00-24:00)を除外
- **注意**: 新運用では NY時間も取引対象。動的スプレッドゲート(`InpMaxSpreadPoints`)で指標時の急拡大を自動防御

### `InpJSTFromBroker` — JSTとブローカー時刻の差
- **既定値**: `6` (現在は夏時間期間のため) ← 2026年4月更新
- **意味**: MT4サーバー時刻に何時間足すとJSTになるか
- **重要**: **夏時間=6 / 冬時間=7**
  - 夏時間期間: 2026年3月29日（日）〜 2026年10月25日（日） → `6`
  - 冬時間期間: 2026年10月26日（月）〜 2027年3月28日（土） → `7`
  - 切替日の夜に必ず手動で書き換え（全チャートのEA設定を更新 or git pullで一括更新）
- **確認方法**: EA起動時にエキスパートタブに以下のログが出る:
  ```
  Time: broker=2026.04.12 21:35 JST=04:35 (offset=6)
  ```
  このJST表記が**実際のPC時計のJST**と一致すればOK。ずれてたら値を調整
- **症状例**: 8:00 JST から取引が始まってしまう
  - 原因: 夏時間なのに `7` のまま → EAが「JST 9:00」と誤認識
  - 対処: `6` に変更

### `InpTradeOnlyEAHrs` — カスタム取引時間帯を有効化
- **既定値**: `true` (新運用方針で有効化)
- **意味**: trueにすると、`InpStartHour`〜`InpEndHour` の時間帯でのみ取引
- **深夜跨ぎ対応**: StartHour > EndHour の場合、翌日EndHourまでを含む
  - 例: Start=9, End=3 → 9:00〜翌3:00 (18時間稼働)

### `InpStartHour` / `InpEndHour` — 取引時間帯
- **既定値**: `9` / `3`
- **運用例**:
  - `9` / `3` (デフォルト): JST 9:00〜翌3:00 稼働（**18時間稼働/6時間休み**）
  - `9` / `15`: 東京時間のみ (6時間)
  - `15` / `24`: ロンドン+NY前半のみ
  - `0` / `24`: 24時間稼働
- **休止時間(3:00〜9:00)の意義**:
  - 早朝の流動性低下・スプレッド拡大帯を回避
  - サマータイム移行時の不安定期を回避
  - EA監視・メンテナンスの空白時間

### `InpVerbose` — 詳細ログ
- **既定値**: `false`
- **意味**: trueにすると全ティックの判定ログをExpertタブに出力
- **注意**: バックテストではONにすると動作が遅くなる。ライブでは診断時のみON

### `InpLogCSV` — 診断CSVログの出力
- **既定値**: `false`
- **意味**: trueにすると `MQL4/Files/TickScalp_<Symbol>.csv` に1イベント1行で診断値を append
- **記録される情報**:
  - ENTRY / EXIT_STOP / EXIT_TRAIL / EXIT_TIMEOUT / DIR (方向変化)
  - 各イベント時の emaGap, emaSlope, hits, tickMove, spread, holdSec, pip 損益
- **CSVカラム** (区切り: `;`):
  ```
  timestamp;symbol;event;side;pip;holdSec;emaGap;emaSlope;hits;tickMove;spread
  ```
- **用途**: エントリ直後の逆行損切り率や、弱い条件帯（例: emaGap < 0.3p）を統計的に特定するため
- **注意**:
  - 通貨ごとに別ファイルになる (例: `TickScalp_USDJPY-cd.csv`)
  - バックテストでは大量に append されるので、分析時に便利だがファイルサイズ要注意
  - 本番運用では**まずONにして1〜2日回し、分析後にOFF**が想定される流れ
- **解析方法**: `fx-scalping/診断ログ解析ガイド.md` に Python/Excel サンプル

## 診断ログの読み方（Expertタブ）

### ENTRY ログ
```
ENTRY BUY @150.123 ticket=456 emaGap=+0.70p slope=+0.30p/bar hits=4/5 tickMove=+0.40p spread=0.2p
```
- `emaGap`: EMA(20)とEMA(60)の距離(pip)。**絶対値が大きいほどトレンドが強い**
- `slope`: EMA(20)の傾き (1バーあたりのpip変化)。**符号が方向と一致していれば健全**
- `hits`: 直近 InpTriggerWindow-1 tick 差分のうち順方向だった数
- `tickMove`: 直近 windowサイズ tick の合計移動量(pip)。**0.1p未満はノイズ疑い**
- `spread`: 発注時スプレッド

### EXIT ログ
```
EXIT[STOP] side=BUY pip=-2.50 peak=+0.30 holdSec=8 ticket=456 (entry: gap=+0.70p slope=+0.30p/bar hits=4/5 move=+0.40p)
```
- `holdSec`: 保有秒数。**5秒以内の損切りは完全な逆行シグナル**
- `peak`: 保有中の最大含み益pip。peakが0の損切り = 一度も浮かなかった
- カッコ内はエントリ時の診断値再出力

### DIR ログ (方向反転時のみ)
```
DIR BUY->SELL [14:23] emaF=150.110 emaS=150.115 gap=-0.05p slope=-0.02p/bar
```
- gapの絶対値が小さい方向反転 (例: 0.05p) は**フラット相場のwhipsaw疑い**
- 反転直後のエントリは逆行しやすい傾向を疑う

## チューニングの優先順位

実運用前に調整するなら、優先度順に:

1. **`InpLots`** — 契約サイズに合わせる(これ間違えるとランク1超過で手数料発生)
2. **`InpJSTFromBroker`** — 冬/夏で切替
3. **`InpMaxSpreadPoints`** — ライブで気配値観察してから調整
4. **`InpStopPip` / `InpPeakActivate` / `InpTrailGap`** — MT4のGA最適化で自動チューニング

## MT4最適化（GA）の推奨レンジ

ストラテジーテスターの「最適化」を有効化し、パラメータ画面で:

| 変数 | Start | Step | Stop |
|---|---:|---:|---:|
| InpStopPip | 1.5 | 0.5 | 3.5 |
| InpPeakActivate | 1.0 | 0.3 | 2.5 |
| InpTrailGap | 0.5 | 0.3 | 2.0 |
| InpTriggerHits | 3 | 1 | 4 |

→ 約200〜300通り × シード1でGenetic Algorithm → 数時間で収束
