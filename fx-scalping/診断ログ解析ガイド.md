# 診断ログ解析ガイド

EA に `InpLogCSV = true` を設定すると `MQL4/Files/TickScalp_<Symbol>.csv` に診断値が記録される。本ガイドはそのCSVを使って **「方向判断の弱点」を実データで特定する** 手順を示す。

## CSV カラム仕様

区切り: **`;`** (セミコロン)
ANSI エンコーディング

| 列 | 名前 | 型 | 記録イベント | 意味 |
|---|---|---|---|---|
| 1 | `timestamp` | `YYYY.MM.DD HH:MM:SS` | 全部 | MT4サーバ時刻 |
| 2 | `symbol` | string | 全部 | 例: `USDJPY-cd` |
| 3 | `event` | string | 全部 | `ENTRY` / `EXIT_STOP` / `EXIT_TRAIL` / `EXIT_TIMEOUT` / `DIR` |
| 4 | `side` | `BUY`/`SELL`/`FLAT` | 全部 | エントリ/退出の方向 |
| 5 | `pip` | float (+/-) | EXIT のみ | 確定pip損益 |
| 6 | `holdSec` | int | EXIT のみ | 保有秒数 |
| 7 | `emaGap` | float (+/-) | 全部 | EMA(20)-EMA(60) の距離 [pip] |
| 8 | `emaSlope` | float (+/-) | 全部 | EMA(20) の傾き [pip/bar] |
| 9 | `hits` | int | ENTRY/EXIT のみ | エントリ時の tick hit 数 |
| 10 | `tickMove` | float (+/-) | ENTRY/EXIT のみ | エントリ時の tick window 総移動 [pip] |
| 11 | `spread` | float | ENTRY/EXIT のみ | エントリ時スプレッド [point] |
| 12 | `m5Gap` | float (+/-) | 全部 | M5 EMA(20)-EMA(60) の距離 [pip] (マルチTF確認用) |

DIR イベントは pip / hits / tickMove / spread が空欄になる（方向変化の記録のみ）。
m5Gap は DIR でも ENTRY/EXIT でも記録される。

## 解析の目的

**ユーザー観測**: エントリー直後に逆行して -2.5p 損切りが多い

→ どういう条件のエントリが逆行しやすいのかを特定し、将来のフィルタ閾値を決める。

## Python による分析サンプル

```python
import pandas as pd
import numpy as np

df = pd.read_csv('TickScalp_USDJPY-cd.csv', sep=';')

# ENTRY と EXIT を ticket順にペアリング (時系列順で隣接)
entries = df[df.event == 'ENTRY'].reset_index(drop=True)
exits   = df[df.event.str.startswith('EXIT')].reset_index(drop=True)

# 長さ一致を確認
assert len(entries) == len(exits), f"entry={len(entries)} exit={len(exits)}"

trades = pd.DataFrame({
    'entry_ts': pd.to_datetime(entries.timestamp),
    'side':     entries.side,
    'emaGap':   entries.emaGap,
    'emaSlope': entries.emaSlope,
    'hits':     entries.hits,
    'tickMove': entries.tickMove,
    'spread':   entries.spread,
    'm5Gap':    entries.m5Gap,
    'exitReason': exits.event.str.replace('EXIT_', ''),
    'pip':      exits.pip,
    'holdSec':  exits.holdSec,
})

# 逆行損切り = STOP で終わったトレード
trades['isLoss']    = trades.exitReason == 'STOP'
trades['isQuickLoss'] = (trades.exitReason == 'STOP') & (trades.holdSec <= 5)

print(f"Total trades: {len(trades)}")
print(f"Win rate: {(trades.pip > 0).mean():.1%}")
print(f"Loss rate: {trades.isLoss.mean():.1%}")
print(f"Quick loss (<=5s): {trades.isQuickLoss.mean():.1%}")

# emaGap 帯ごとの勝率
bins = [0, 0.2, 0.5, 1.0, 999]
trades['gapBin'] = pd.cut(trades.emaGap.abs(), bins, right=False)
print("\n=== emaGap vs 勝率 ===")
print(trades.groupby('gapBin').agg(
    count=('pip','size'),
    win_rate=('pip', lambda x: (x > 0).mean()),
    avg_pip=('pip','mean'),
    quick_loss_rate=('isQuickLoss','mean'),
))

# hits 数ごとの勝率
print("\n=== hits vs 勝率 ===")
print(trades.groupby('hits').agg(
    count=('pip','size'),
    win_rate=('pip', lambda x: (x > 0).mean()),
    avg_pip=('pip','mean'),
))

# tickMove 帯ごとの勝率
trades['moveBin'] = pd.cut(trades.tickMove.abs(), [0, 0.1, 0.3, 0.5, 999], right=False)
print("\n=== tickMove vs 勝率 ===")
print(trades.groupby('moveBin').agg(
    count=('pip','size'),
    win_rate=('pip', lambda x: (x > 0).mean()),
    avg_pip=('pip','mean'),
))

# emaSlope と方向の一致度
trades['slopeAgree'] = np.sign(trades.emaSlope) == trades.side.map({'BUY':1,'SELL':-1})
print("\n=== emaSlope の向きと方向の一致 vs 勝率 ===")
print(trades.groupby('slopeAgree').agg(
    count=('pip','size'),
    win_rate=('pip', lambda x: (x > 0).mean()),
    avg_pip=('pip','mean'),
))

# M5 Gap 帯ごとの勝率 (マルチTF 確認が効いているかの検証)
trades['m5GapBin'] = pd.cut(trades.m5Gap.abs(), [0, 0.3, 1.0, 3.0, 999], right=False)
print("\n=== M5 Gap 絶対値 vs 勝率 ===")
print(trades.groupby('m5GapBin').agg(
    count=('pip','size'),
    win_rate=('pip', lambda x: (x > 0).mean()),
    avg_pip=('pip','mean'),
))

# M1 と M5 の整合性 (InpUseM5Confirm=true なら全エントリで一致するはず)
trades['m5Agree'] = np.sign(trades.m5Gap) == trades.side.map({'BUY':1,'SELL':-1})
print("\n=== M5 方向一致 vs 勝率 ===")
print(trades.groupby('m5Agree').agg(
    count=('pip','size'),
    win_rate=('pip', lambda x: (x > 0).mean()),
    avg_pip=('pip','mean'),
))
```

## Excel での簡易分析

1. CSV を開く（区切りは `;`）
2. データ → ピボットテーブル
3. 行: `event` でフィルタ → EXIT_STOP のみ
4. 値: `pip` の平均 / `holdSec` の平均
5. 列: `emaGap` を 0.2 刻みでグループ化 (`=FLOOR(G2, 0.2)` 列を作る)
6. 「どの emaGap 帯で holdSec が極端に短い（= 即逆行）」かを目視

## 読み解きの指針

弱条件の見つけ方:

- **勝率が 45% 未満** の条件帯は切るべき
- **holdSec <= 5s の損切り率が 20% 超** の条件帯は「逆行トラップ」
- **emaGap < 0.2p** で loss rate が高いなら → `InpMinEmaGapPip = 0.3` が妥当
- **tickMove < 0.1p** で loss rate が高いなら → ノイズ発火、`InpMinTickMovePip = 0.2` が妥当
- **emaSlope と方向が不一致** で loss rate が高いなら → slope agreement filter を追加
